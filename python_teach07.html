主題一：串列生成式 (List Comprehension)

串列生成式是最常見也最基礎的生成式，其目標是快速生成一個新的串列。 它提供了一種優雅、緊湊的單行語法，來取代傳統的 for 迴圈與 `.append()` 操作。

A. 核心概念與規則

  • 基礎語法： new_list = [expression for item in iterable]
  • 篩選語法： new_list = [expression for item in iterable if condition]
  • 條件表達式語法： new_list = [expr_if_true if condition else expr_if_false for item in iterable]
  • 關鍵組成：
    • expression: 對於 iterable 中的每一個 item 進行運算或處理的表達式。
    • item: 從 iterable 中逐一取出的元素。
    • iterable: 任何可迭代的物件，如 list, tuple, range, string 等。

B. 範例程式碼

範例 1：基本轉換 (平方運算)

# 傳統 for 迴圈寫法
numbers = [1, 2, 3, 4, 5]
squares = []
for n in numbers:
    squares.append(n**2)
print(squares)  # >> [1, 4, 9, 16, 25]

# 使用串列生成式
squares_comp = [n**2 for n in numbers]
print(squares_comp)  # >> [1, 4, 9, 16, 25]

C. 程式碼細緻拆解

  1. [n**2 for n in numbers] 這一行程式碼精確地表達了其意圖：「為 numbers 中的每一個 n，計算其平方 (n**2)，並將所有結果收集成一個新的串列」。

範例 2：加入條件篩選 (if)

numbers = [1, 2, 3, 4, 5, 6, 7, 8]
# 只挑選出偶數，並計算其平方
even_squares_comp = [n**2 for n in numbers if n % 2 == 0]
print(even_squares_comp)  # >> [4, 16, 36, 64]

C. 程式碼細緻拆解

  1. if n % 2 == 0 這個條件過濾了 `numbers` 串列。
  2. 只有當 `n` 是偶數時，`n**2` 這個表達式才會被執行並加入新的串列中。

範例 3：加入條件表達式 (if-else)

numbers = [1, 2, 3, 4, 5]
# 將數字串列轉換為字串串列，偶數標記為 'even'，奇數標記為 'odd'
labels_comp = ['even' if n % 2 == 0 else 'odd' for n in numbers]
print(labels_comp)  # >> ['odd', 'even', 'odd', 'even', 'odd']

C. 程式碼細緻拆解

  1. 注意 `if-else` 結構是放在 `expression` 的位置，與單純的 `if` 篩選位置不同。
  2. 'even' if n % 2 == 0 else 'odd': 對於 `numbers` 中的每個 `n`，如果 `n` 是偶數，則表達式的值為 `'even'`，否則為 `'odd'`。

範例 4：巢狀迴圈

matrix = [[1, 2, 3], [4, 5, 6], [7, 8, 9]]
# 將一個二維串列扁平化為一維串列
flattened_comp = [num for row in matrix for num in row]
print(flattened_comp)  # >> [1, 2, 3, 4, 5, 6, 7, 8, 9]

C. 程式碼細緻拆解

  1. for row in matrix 是外層迴圈，for num in row 是內層迴圈，其順序與傳統巢狀迴圈寫法一致。

主題二：字典生成式 (Dictionary Comprehension)

與串列生成式類似，字典生成式讓我們能快速地創建字典。

A. 核心概念與規則

  • 基礎語法： new_dict = {key_expression: value_expression for item in iterable}
  • 功能： 可用於從可迭代物件創建字典，或基於現有字典進行篩選與轉換。

B. 範例程式碼

範例 5：創建數字與其平方的對應字典

# 使用字典生成式
squares_dict_comp = {i: i**2 for i in range(1, 6)}
print(squares_dict_comp)  # >> {1: 1, 2: 4, 3: 9, 4: 16, 5: 25}

範例 6：從既有字典篩選或轉換

prices = {'apple': 50, 'banana': 30, 'laptop': 30000, 'mouse': 150}
# 建立一個只包含價格大於 100 的新字典
expensive_items = {item: price for item, price in prices.items() if price > 100}
print(expensive_items)  # >> {'laptop': 30000, 'mouse': 150}

C. 程式碼細緻拆解

  1. .items() 方法會將字典的鍵值對以元組 (key, value) 的形式回傳，讓我們可以在生成式中同時操作鍵和值。
  2. if price > 100 篩選出符合條件的鍵值對來建立新字典。

主題三：集合生成式 (Set Comprehension)

集合生成式的語法與字典生成式非常相似，但它只包含一個 expression。 集合最重要的特性是其元素是唯一不重複的。

A. 核心概念與規則

  • 基礎語法： new_set = {expression for item in iterable}
  • 特性： 生成的結果會自動去除重複的元素。

B. 範例程式碼

範例 7：從包含重複元素的串列中建立唯一值的集合

numbers = [1, 2, 2, 3, 4, 4, 4, 5]
# 從一個串列中提取所有數字的平方，並確保結果中沒有重複值
unique_squares = {n**2 for n in numbers}
print(unique_squares)  # >> {1, 4, 9, 16, 25}

C. 程式碼細緻拆解

  1. 儘管輸入的 `numbers` 串列中有重複的 2 和 4，但它們的平方 4 和 16 在最終的集合中只會出現一次。

主題四：生成器表達式 (Generator Expression)

使用 () 的生成式語法並不會立即建立一個元組，而是會建立一個生成器物件 (generator object)。

A. 核心概念與規則

  • 語法： generator_object = (expression for item in iterable)
  • 延遲計算 (Lazy Evaluation)： 生成器不會一次性在記憶體中產生所有的結果，而是在每次被請求時才「即時」計算並回傳下一個值。
  • 記憶體效率： 這種特性使其在處理大量資料時極具記憶體效率。
  • 轉換為元組： 若要獲得一個元組，必須明確地使用 tuple() 建構函式。

B. 範例程式碼

範例 8：生成器表達式與元組轉換

numbers = [1, 2, 3, 4, 5]
# 這是一個生成器表達式
squared_gen = (n**2 for n in numbers)
print(squared_gen)  # >> <generator object <genexpr> at 0x...>

# 將生成器物件轉換為元組
squared_tuple = tuple(squared_gen)
print(squared_tuple)  # >> (1, 4, 9, 16, 25)

C. 程式碼細緻拆解

  1. (n**2 for n in numbers) 立即返回一個生成器物件，程式碼並未在此時進行任何計算。
  2. 當 `tuple()` 函式被呼叫時，它會迭代 `squared_gen` 生成器，逐一取出計算後的值 (1, 4, 9...)，最終建構成一個完整的元組。

D. 前瞻性觀點

Python 不直接提供「元組生成式」是基於其設計哲學。元組是不可變的(immutable)，一次性從一個完整的可迭代物件 (如生成器) 創建，是更符合其特性的作法。 使用生成器表達式搭配 `tuple()` 的兩步驟流程，在處理龐大資料集時，效能和記憶體使用上更具優勢。

結論與最佳實踐

對於 Python 開發者而言，熟練掌握生成式是邁向編寫更高效、更具可讀性程式碼的關鍵一步。 在機器學習與資料科學領域，生成式是進行大規模資料集預處理與轉換的強力工具。

語法速查表

類型 語法 主要用途 特性
串列生成式 [expr for item in iter] 創建新的串列 靈活、常用，可包含複雜邏輯
字典生成式 {key: val for item in iter} 創建新的字典 高效建構鍵值對
集合生成式 {expr for item in iter} 創建新的集合 自動處理重複元素，確保唯一性
生成器表達式 (expr for item in iter) 創建生成器物件 延遲計算，記憶體效率極高

最佳實踐建議

  • 優先使用： 當您需要根據現有可迭代物件創建新的串列、字典或集合時，應優先考慮使用生成式。
  • 保持簡潔： 如果生成式變得過於複雜（例如超過兩層的巢狀迴圈），為了可讀性與可維護性，退回到傳統的 for 迴圈可能是更好的選擇。
  • 善用生成器： 在處理可能消耗大量記憶體的龐大資料流時，請務必優先考慮使用生成器表達式，以最小化資源佔用。
