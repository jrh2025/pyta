主題一：什麼是繼承 (Inheritance)？

繼承是一種物件導向的概念，它允許一個類別（稱為 子類別 Subclass）獲取另一個類別（稱為 父類別 Superclass）的屬性和方法。這種關係通常被描述為「is-a」（是一種）的關係。例如，一個「學生」是一種「人」，因此「學生」類別可以繼承自「人」類別。

A. 範例程式碼：基本的繼承關係

# 定義父類別：Person
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age
        print(f"(父類別 Person 的 __init__ 已被呼叫)")

    def display_info(self):
        print(f"姓名: {self.name}, 年齡: {self.age}")

# 定義子類別：Student，它繼承自 Person
class Student(Person):
    # Student 類別目前沒有定義任何自己的方法或屬性
    # 但它會自動擁有 Person 的一切
    pass

# --- 實際操作 ---
# 建立一個 Student 物件
student1 = Student("王大明", 20)

# 呼叫繼承自 Person 類別的 display_info 方法
student1.display_info()

B. 程式碼細緻拆解

  1. class Student(Person):: 這行程式碼定義了 `Student` 類別，並在括號中指定 `Person` 作為其父類別。
  2. `pass`: 關鍵字表示 `Student` 類別的主體是空的，它沒有添加任何新的屬性或方法。
  3. student1 = Student("王大明", 20): 即使 `Student` 類別沒有定義自己的 `__init__` 方法，它會自動呼叫父類別 `Person` 的 `__init__` 方法來初始化物件。
  4. student1.display_info(): `student1` 物件可以直接呼叫 `Person` 類別中定義的 `display_info` 方法，這完美展示了程式碼的複用。

主題二：覆寫方法 (Method Overriding)

子類別雖然繼承了父類別的所有方法，但有時父類別的方法行為不完全符合子類別的需求。這時，子類別可以重新定義一個與父類別同名的方法，這個過程稱為 覆寫 (Overriding)。

A. 範例程式碼：覆寫 display_info 方法

class Student(Person):
    def __init__(self, name, age, student_id):
        # ... (初始化暫略，詳見下一節)
        super().__init__(name, age)
        self.student_id = student_id

    # --- 覆寫父類別的 display_info 方法 ---
    def display_info(self):
        # 這裡我們完全重新定義了方法的內容
        print(f"姓名: {self.name}, 年齡: {self.age}, 學號: {self.student_id}")

# --- 實際操作 ---
student2 = Student("陳小美", 21, "S654321")
print("\nStudent 物件的資訊 (使用覆寫後的方法)：")
student2.display_info()

B. 程式碼細緻拆解

  1. `def display_info(self):` 在 `Student` 類別中再次出現，代表我們要覆寫來自父類別的同名方法。
  2. 當 student2.display_info() 被呼叫時，Python 會優先使用 `Student` 類別中定義的版本，因為它更「具體」。

主題三：使用 super() 呼叫父類別方法

為了避免在子類別中重複撰寫父類別已有的功能，我們可以使用 super() 函式。super() 讓我們可以從子類別中明確地呼叫父類別的方法，這是繼承中的最佳實踐。

A. 範例程式碼：使用 super() 優化

class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age
    def display_info(self):
        print(f"姓名: {self.name}, 年齡: {self.age}", end="")

class Student(Person):
    def __init__(self, name, age, student_id):
        # 使用 super() 呼叫父類別的 __init__
        super().__init__(name, age)
        # 子類別只需處理自己特有的屬性
        self.student_id = student_id

    # --- 使用 super() 優化覆寫方法 ---
    def display_info(self):
        # 首先，呼叫父類別的 display_info 方法
        super().display_info()
        # 然後，子類別只需添加自己獨特的部分
        print(f", 學號: {self.student_id}")

student3 = Student("李四", 22, "S987654")
student3.display_info()

B. 程式碼細緻拆解

  1. super().__init__(name, age): 這行程式碼明確地呼叫父類別 (`Person`) 的 `__init__` 方法，讓它去完成 `name` 和 `age` 的初始化，避免了程式碼重複。
  2. super().display_info(): 在子類別的 `display_info` 中，我們先呼叫父類別的同名方法來處理共通的輸出部分（姓名和年齡）。
  3. 接著，子類別只需 `print` 自己特有的資訊（學號），實現了功能的擴充而非完全重寫。

主題四：多層繼承 (Multilevel Inheritance)

繼承關係可以形成一個鏈條。一個類別可以繼承自另一個子類別，例如：`Person -> Student -> PhDStudent`。

class PhDStudent(Student):
    def __init__(self, name, age, student_id, research_topic):
        # 呼叫其父類別 (Student) 的 __init__
        super().__init__(name, age, student_id)
        self.research_topic = research_topic

    def display_info(self):
        super().display_info()
        print(f", 研究主題: {self.research_topic}")

phd1 = PhDStudent("林博士", 28, "P112233", "生成式AI")
phd1.display_info()
`phd1` 物件同時擁有 `Person`、`Student` 和 `PhDStudent` 三個類別的特性。`super()` 在此處會呼叫其直接父類別 `Student` 的方法。

主題五：多重繼承 (Multiple Inheritance)

Python 允許一個子類別同時繼承多個父類別，這在需要組合不同功能時非常有用。

class Worker:
    def __init__(self, job_title):
        self.job_title = job_title
    def work(self):
        print(f"正在以 {self.job_title} 的身份工作...")

# WorkingStudent 同時繼承 Student 和 Worker
class WorkingStudent(Student, Worker):
    def __init__(self, name, age, student_id, job_title):
        # 當有多個父類別時，需要分別初始化
        Student.__init__(self, name, age, student_id)
        Worker.__init__(self, job_title)

ws1 = WorkingStudent("張三", 23, "S556677", "研究助理")
ws1.display_info()  # 來自 Student
ws1.work()          # 來自 Worker
警示：多重繼承雖然強大，但也可能引發「鑽石問題」。Python 使用一套稱為 方法解析順序 (MRO) 的演算法來解決此問題。對於初學者，建議謹慎使用。

主題六：私有方法與屬性 (Name Mangling)

在 Python 中，可以在名稱前加上兩個底線 (__) 來建立「私有」成員。這並非真正的私有，而是一種稱為「名稱修飾 (Name Mangling)」的機制，目的是避免子類別無意中覆寫了父類別的內部方法。

class Person:
    def __init__(self, name, ssn):
        self.name = name
        self.__ssn = ssn  # 私有屬性

    # 私有方法
    def __get_ssn(self):
        return self.__ssn

    def access_sensitive_info(self):
        # 類別內部可以自由呼叫自己的私有方法
        ssn = self.__get_ssn()
        print(f"{self.name} 的 SSN 後四碼為 {ssn[-4:]}")

p = Person("秘密客", "ABC-12345")
p.access_sensitive_info() # 成功

try:
    print(p.__ssn)
except AttributeError as e:
    print(f"\n存取私有屬性失敗: {e}")

結論與前瞻性觀點

繼承是物件導向程式設計中實現抽象化和程式碼複用的核心工具。透過建立清晰的類別層次，我們可以減少重複程式碼、提高可維護性，並建立更精確反映真實世界關係的邏輯模型。

在 PyTorch 中建立一個神經網路模型時，您的模型類別通常會繼承自 `torch.nn.Module`，從而自動獲得模型訓練、參數管理等所有底層功能。理解繼承，是您從腳本編寫者邁向軟體架構師的必經之路。
