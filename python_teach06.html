第一部分：進階迴圈技巧與迭代模式

在基礎篇之外，Python 提供了多種強大的工具，能讓迴圈的撰寫更為精煉與高效。

1.1 列表生成式 (List Comprehensions): 簡潔與效率的極致

列表生成式是 Python 最具代表性的特色之一。它允許您用一行程式碼來取代傳統的多行 for 迴圈，以建立新的列表。這不僅讓程式碼更簡潔，在許多情況下執行效率也更高。

傳統寫法 vs. 列表生成式

# 【傳統 for 迴圈寫法】
squares = []
for i in range(10):
    squares.append(i * i)
print(f"傳統寫法結果: {squares}")

# 【使用列表生成式】
# 語法: [expression for item in iterable]
squares_comp = [i * i for i in range(10)]
print(f"列表生成式結果: {squares_comp}")

列表生成式 (含條件判斷)

# 【範例：只計算偶數的平方】
# 傳統寫法
even_squares = []
for i in range(10):
    if i % 2 == 0:
        even_squares.append(i * i)
print(f"傳統寫法 (含條件): {even_squares}")

# 列表生成式 (含 if 條件)
# 語法: [expression for item in iterable if condition]
even_squares_comp = [i * i for i in range(10) if i % 2 == 0]
print(f"列表生成式 (含條件): {even_squares_comp}")
專業觀點：在資料科學與機器學習的特徵工程中，列表生成式被廣泛用於對資料集進行快速轉換、清洗與篩選，是每一位 Python 資料分析師必備的技能。

1.2 enumerate(): 同時獲取索引與值

當遍歷一個序列時，有時我們既需要元素的值，也需要該元素的索引。傳統作法是自行維護一個計數器，但 Python 提供了更優雅的 enumerate() 函式。

# 【範例：印出學生成績及其排名】
student_scores = [85, 92, 78, 95, 88]
print("\n【使用 enumerate() 獲取索引和值】")

# Pythonic 寫法：使用 enumerate()
# enumerate() 會回傳一個包含 (索引, 值) 的元組 (tuple)
for index, score in enumerate(student_scores):
    print(f"排名 {index + 1}: 分數 {score}")

1.3 zip(): 並行遍歷多個序列

若您需要同時遍歷兩個或多個序列，zip() 函式可以將這些序列「壓縮」在一起，讓您在單一 for 迴圈中並行迭代。

# 【範例：將學生姓名與成績配對】
students = ["Alice", "Bob", "Charlie", "David"]
scores = [85, 92, 78, 95]
print("\n【使用 zip() 並行遍歷】")

# zip() 會將對應位置的元素打包成元組
for student, score in zip(students, scores):
    print(f"學生 {student} 的成績是 {score} 分。")
# 注意：如果序列長度不同，zip() 會在最短的序列結束時停止。
專業觀點：在監督式學習中，特徵資料 (X) 與標籤資料 (y) 通常是分開的陣列或列表。zip(X, y) 是將特徵與其對應標籤配對進行處理的標準模式。

1.4 迴圈中的 else 區塊

這是一個較不為人知但非常實用的語法。for 或 while 迴圈可以帶有一個 else 區塊。這個 else 區塊的執行時機是：當迴圈正常執行完畢 (即沒有被 break 陳述式中斷) 時。這非常適合用在「搜尋」情境中。

# 【範例：在列表中搜尋一個質數，如果沒找到就發出通知】
numbers = [6, 15, 21, 33, 55]
prime_to_find = 17  # 一個不存在於列表中的質數
print("\n【在 for...else 結構中搜尋】")

for num in numbers:
    if num == prime_to_find:
        print(f"找到了目標質數: {num}")
        break  # 一旦找到，就中斷迴圈
else:
    # 如果 for 迴圈從未被 break，代表搜尋完畢但沒找到
    print(f"列表中未找到質數 {prime_to_find}。")

這個結構避免了額外使用一個旗標變數 (e.g., found = False) 來判斷迴圈是否被中斷，讓程式碼更乾淨、意圖更明確。

第二部分：進階例外處理

穩健的系統不僅要能處理預期的錯誤，更要能定義自己的錯誤類型，並以最安全的方式管理資源。

2.1 使用 with 陳述式自動管理資源

Python 提供了 with 陳述式，又稱為「內容管理器 (Context Manager)」，能夠更優雅且安全地處理需要設定 (setup) 與清理 (teardown) 的資源，如檔案、網路連線、資料庫交易等。

# 【範例：使用 with 陳述式讀取檔案】
file_path = "config.txt"
print(f"\n【使用 'with' 陳述式自動管理資源】")

try:
    # with 會自動處理 file.close()
    # 無論 with 區塊內發生任何錯誤或正常結束，Python 都會確保檔案被關閉
    with open(file_path, 'r', encoding='utf-8') as file:
        print(f"檔案 '{file_path}' 已成功開啟。")
        content = file.read()
        print("--- 檔案內容 ---")
        print(content)
        print("------------------")
except FileNotFoundError:
    print(f"錯誤：檔案 '{file_path}' 不存在。")
except Exception as e:
    print(f"處理檔案時發生了未預期的錯誤: {e}")

print("'with' 區塊結束後，檔案會被自動關閉，無需手動呼叫 file.close()。")
最佳實踐：永遠優先使用 with 陳述式來處理檔案和網路連線等資源，而非手動編寫 try...finally。

2.2 自訂例外 (Custom Exceptions): 建立語意化的錯誤

在開發大型應用或函式庫時，建立自己的例外類型能大幅提升程式碼的可維護性與清晰度。自訂例外是透過繼承 Python 的 Exception 基底類別來建立的。

# --- 1. 定義自訂例外類別 ---
class DataValidationError(Exception):
    """當輸入資料不符合業務邏輯時拋出的例外。"""
    pass

class InsufficientDataError(DataValidationError):
    """當資料筆數不足以進行分析時拋出的例外。"""
    pass

# --- 2. 在函式中使用自訂例外 ---
def process_data(data_points):
    print(f"\n【在函式中拋出與處理自訂例外】")
    print(f"正在處理資料: {data_points}")
    
    if len(data_points) < 3:
        raise InsufficientDataError("輸入資料不足，至少需要 3 個資料點。")
    
    for point in data_points:
        if point <= 0:
            raise DataValidationError(f"資料點 '{point}' 無效，必須為正數。")
            
    print("資料驗證通過，處理完成。")

# --- 3. 捕捉自訂例外 ---
try:
    # process_data([10, 20])  # 將觸發 InsufficientDataError
    process_data([10, 20, -5, 30])  # 將觸發 DataValidationError
except InsufficientDataError as e:
    print(f"捕捉到資料不足錯誤: {e}")
except DataValidationError as e:
    print(f"捕捉到資料驗證錯誤: {e}")

2.3 raise 與例外鏈結 (Exception Chaining)

有時，在捕捉到一個低階例外後，您可能想將其包裝成一個更具業務意義的高階例外再向上拋出。Python 3 引入了「例外鏈結」(raise...from...) 來解決這個問題，它能保留原始的錯誤追蹤資訊，讓除錯更為方便。

# 【範例：使用 'raise from' 進行例外鏈結】
class ReportGenerationError(Exception):
    """報告生成時的頂層錯誤。"""
    pass

def generate_report(data_file):
    print("\n【使用 'raise from' 保留原始錯誤堆疊】")
    try:
        with open(data_file, 'r') as f:
            content = f.read()
            if not content:
                raise KeyError("設定檔中缺少必要的鍵值。")
        return "報告生成成功！"
    except (IOError, KeyError) as e:
        # 將底層錯誤包裝成高層業務例外，並用 'from' 保留原始例外資訊
        raise ReportGenerationError("無法生成報告，因來源資料有問題。") from e

# --- 呼叫端 ---
try:
    generate_report("non_existent_file.txt")
except ReportGenerationError as e:
    print(f"捕捉到高階錯誤: {e}")
    # __cause__ 屬性會保留原始例外
    print(f"根本原因 (Root Cause): {type(e.__cause__).__name__}: {e.__cause__}")
