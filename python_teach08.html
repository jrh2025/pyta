主題一：什麼是類別 (Class)？

您可以將 類別 想像成一張「藍圖」或一個「模板」。它定義了一個事物的共同結構和行為，但它本身並不是那個事物。

例如，一張「汽車」的藍圖會描述汽車應具備的屬性 (attributes)，如品牌、顏色、型號等；同時，它也會定義汽車應有的方法 (methods) 或功能，如啟動、加速、煞車等。這張抽象的「汽車」藍圖，就是一個類別。

主題二：什麼是物件 (Object)？

物件 (Object) 則是根據類別這張「藍圖」所創造出來的「實體 (instance)」。每一輛實際存在、可以駕駛的汽車都是一個物件。

  • 一輛紅色的 Toyota Corolla 是一個汽車物件。
  • 一輛黑色的 Tesla Model S 是另一個汽車物件。

這兩個物件都是根據「汽車」類別所建立的，因此它們都擁有品牌、顏色等屬性，以及加速、煞車等方法。然而，它們是獨立存在的個體，每個物件的屬性值可以是不同的。

主題三：建立類別 (Class)

一個類別通常包含以下幾個關鍵部分：

  • class 關鍵字：用於宣告一個類別。
  • 類別名稱：按照慣例，使用大駝峰式命名法 (PascalCase)，例如 `Student`。
  • __init__ 方法：一個特殊的「初始化方法」或「建構式」，當新物件被建立時會自動被呼叫，用於設定初始屬性。
  • self 參數：在類別的方法中，代表「物件本身」，用於存取該物件的屬性與方法。
  • 屬性 (Attributes)：儲存物件狀態的變數，例如 `self.name`。
  • 方法 (Methods)：定義在類別內部的函式，用於執行物件的行為。

A. 範例程式碼：Student 類別

# 使用 class 關鍵字定義 Student 類別
class Student:
    """
    這是一個代表學生的類別。
    它包含學生的基本資訊和可以執行的操作。
    """

    # __init__ 是初始化方法 (建構式)
    # 當 Student 物件被建立時，此方法會自動執行
    def __init__(self, name: str, student_id: str):
        """初始化一個新的 Student 物件。"""
        # --- 屬性 (Attributes) ---
        # 將傳入的 name 和 student_id 指派給物件的屬性
        self.name = name
        self.student_id = student_id
        self.courses = []  # 初始化一個空串列來存放修課紀錄

    # --- 方法 (Methods) ---
    def add_course(self, course_name: str):
        """為學生加入一門修習的課程。"""
        self.courses.append(course_name)
        print(f"課程 '{course_name}' 已成功加入 {self.name} 的課表。")

    def display_info(self):
        """顯示學生的完整資訊。"""
        print("--------------------")
        print(f"學生姓名: {self.name}")
        print(f"學生學號: {self.student_id}")
        if self.courses:
            print(f"已選課程: {', '.join(self.courses)}")
        else:
            print("已選課程: 尚未選課")
        print("--------------------")

B. 程式碼細緻拆解

  1. class Student:: 定義了一個名為 `Student` 的新類別。
  2. def __init__(self, name, student_id):: 定義了建構式。當我們寫 `Student(...)` 時，這個方法會被呼叫。`name` 和 `student_id` 是建立物件時必須提供的參數。
  3. self.name = name: 將傳入的 `name` 參數值，儲存為該物件的一個屬性，名為 `name`。
  4. self.courses = []: 為每個新建立的學生物件初始化一個空的 `courses` 串列屬性，用於記錄課程。
  5. def add_course(self, course_name):: 定義了一個方法。`self` 讓這個方法可以存取呼叫它的那個物件的屬性 (例如 `self.courses` 和 `self.name`)。
至此，我們僅僅是定義了「藍圖」，還沒有任何實際的學生存在。

主題四：從同一個類別建立多個物件

現在，我們可以使用 `Student` 類別來建立多個學生物件。建立物件的語法類似於呼叫函式。這個過程稱為「實例化 (Instantiate)」。

A. 範例程式碼

# 根據 Student 類別，建立第一個學生物件 (實體)
student1 = Student("王大明", "S123456")

# 建立第二個學生物件，這是另一個完全獨立的實體
student2 = Student("陳小美", "S654321")

# 呼叫各自物件的方法
student1.add_course("線性代數")
student1.add_course("微積分")
student2.add_course("Python程式設計")

# 顯示每個物件的資訊，可以看到他們的資料是獨立的
print("\n顯示所有學生資訊：")
student1.display_info()
student2.display_info()

B. 程式碼細緻拆解

  1. student1 = Student(...): 呼叫 `Student` 類別的建構式，建立一個新的物件。`"王大明"` 和 `"S123456"` 會被傳遞給 `__init__` 方法的 `name` 和 `student_id` 參數。
  2. student1 和 student2 是 `Student` 類別的兩個不同實例。
  3. 對 `student1` 操作（如 `add_course`）完全不會影響到 `student2`，反之亦然。這就是物件導向封裝性的體現。

主題五：覆寫屬性

預設情況下，物件的屬性是公開的，可以直接從外部存取和修改。

A. 範例程式碼

print(f"student1 的原始姓名: {student1.name}")

# 使用點記法 (dot notation) 直接存取並覆寫屬性值
student1.name = "王小明"

print(f"student1 的更新後姓名: {student1.name}")
student1.display_info()
這種直接存取的特性雖然方便，但也帶來風險。例如，某些重要屬性（如學號）一旦設定後，我們可能不希望它被輕易更改。

主題六：使用 @property 建立唯讀屬性

為了保護某些屬性不被外部隨意修改，同時又希望外部可以讀取它，我們可以使用 `@property` 裝飾器。這是一種更 Pythonic 的作法，讓我們可以將一個方法包裝得像一個屬性一樣被存取。

A. 實作步驟

  1. 將原始屬性名稱前加上一個底線 `_` (例如 `_student_id`)。這是一個慣例，向其他開發者表示這是一個「內部」或「受保護」的屬性。
  2. 定義一個與公開屬性同名的方法 (例如 `student_id`)，並在上方加上 `@property` 裝飾器。此方法僅回傳內部屬性的值。

B. 範例程式碼：將 student_id 設定為唯讀

class Student:
    def __init__(self, name: str, student_id: str):
        self.name = name
        # 1. 將 student_id 重新命名為 _student_id
        self._student_id = student_id
        self.courses = []

    # 2. 使用 @property 裝飾器定義一個 "getter" 方法
    @property
    def student_id(self):
        """取得學生的學號 (此屬性為唯讀)。"""
        return self._student_id
    
    # ... 其他方法 ...

# 建立一個新物件
student3 = Student("李四", "S987654")

# -- 讀取屬性 --
print(f"成功讀取學號: {student3.student_id}")

# -- 試圖寫入屬性 --
try:
    student3.student_id = "S000000"
except AttributeError as e:
    print(f"\n試圖修改學號失敗，發生錯誤: {e}")

C. 程式碼細緻拆解

  1. self._student_id = student_id: 實際的學號資料儲存在 `_student_id` 這個「內部」屬性中。
  2. @property: 這個裝飾器告訴 Python，`def student_id(self):` 這個方法可以用來「讀取」名為 `student_id` 的屬性。
  3. 當我們存取 `student3.student_id` 時，Python 會自動呼叫這個被裝飾的方法並返回 `self._student_id` 的值。
  4. 因為我們沒有定義相對應的 "setter" 方法，所以任何試圖對 `student3.student_id` 賦值的操作都會引發 `AttributeError`，從而確保了資料的完整性與安全性。

結論與前瞻性觀點

類別與物件是現代軟體開發的基石。透過將資料與功能封裝在物件中，我們能夠提高程式碼的模組性、降低複雜度並增強可維護性。

在機器學習領域，從資料預處理器、模型架構到訓練流程，所有複雜的組件幾乎都是以類別的形式來建構的。熟練掌握類別的定義與使用，將為您後續深入學習高階框架（如 Scikit-learn, PyTorch, TensorFlow）奠定堅實的基礎。
