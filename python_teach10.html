主題一：import 的基本語法

在 Python 中，每一個 .py 檔案都可以被視為一個模組。模組讓我們能將程式碼組織在獨立的檔案中，並在需要時載入使用。

A. 方法一：import module_name (推薦)

這是最直接的匯入方式，會將整個模組載入，您需要透過 module_name. 的前綴來存取其中的內容。這種方式可讀性極高，能有效避免命名衝突。

# 匯入整個 math 模組
import math

# 必須使用 'math.' 作為前綴來存取其內容
pi_value = math.pi
result = math.sqrt(16)

print(f"圓周率 (pi) 的值是: {pi_value}")
print(f"16 的平方根是: {result}")

B. 方法二：from module_name import name1, name2

如果您只需要模組中的少數幾個特定功能，可以使用此語法將它們直接匯入到當前的命名空間，使用時不需加上模組前綴。

# 從 math 模組中，只匯入 pi 這個變數和 sqrt 這個函式
from math import pi, sqrt

# 現在可以直接使用 pi 和 sqrt，不需前綴
pi_value = pi
result = sqrt(16)

print(f"圓周率 (pi) 的值是: {pi_value}")
print(f"16 的平方根是: {result}")

C. 方法三：from module_name import * (不推薦)

強烈不建議在正式的專案程式碼中使用此方法。它會污染當前的命名空間，您將無法輕易追蹤變數來源，且極易引發命名衝突。

主題二：使用 as 替模組或函式新增別名

當模組名稱過長，或為了遵循社群慣例時，我們可以使用 as 關鍵字為匯入的模組或函式指定一個較短的別名。

# 1. 為模組設定別名 (在資料科學領域非常常見)
import numpy as np
import pandas as pd

# 使用別名 'np' 來操作 numpy 模組
my_array = np.array([1, 2, 3])
print(f"Numpy array: {my_array}")

# 2. 為特定函式設定別名
from math import factorial as fact

# 使用別名 'fact' 來呼叫 factorial 函式
print(f"5 的階乘是: {fact(5)}")

主題三：建立自己的模組

任何一個 .py 檔案都可以作為模組被其他檔案匯入。

A. 範例程式碼：建立與使用模組

首先，建立一個名為 my_utils.py 的檔案：

# 檔案名稱: my_utils.py
PI = 3.14159265

def add(x, y):
    """回傳兩數之和"""
    return x + y

class Circle:
    def __init__(self, radius):
        self.radius = radius
    def area(self):
        """計算圓面積"""
        return PI * (self.radius ** 2)

在同一個資料夾下，建立 main.py 來匯入並使用它：

# 檔案名稱: main.py
# 匯入我們自己建立的 my_utils 模組
import my_utils

# 使用模組中的變數和函式
sum_result = my_utils.add(10, 5)
print(f"10 + 5 = {sum_result}")

# 建立模組中的類別實體
c = my_utils.Circle(10)
print(f"半徑為 10 的圓面積是: {c.area()}")

B. if __name__ == "__main__" 的重要性

這個條件判斷式是建立可複用模組的關鍵實踐。它讓我們可以在模組檔案中加入僅供測試或獨立執行時才運作的程式碼。

  • 如果檔案是直接被執行 (例如 python my_utils.py)，其 __name__ 變數的值會是 "__main__"。
  • 如果檔案是作為模組被匯入，其 __name__ 變數的值會是模組的名稱 (例如 "my_utils")。

修改 my_utils.py 加入測試區塊：

# ... (函式與類別定義不變) ...

# --- 測試區塊 ---
# 這段程式碼只有在 my_utils.py 被直接執行時才會被觸發
if __name__ == "__main__":
    print("正在直接執行 my_utils.py 檔案進行測試...")
    test_sum = add(50, 50)
    print(f"測試 add(50, 50)，結果為: {test_sum}")

# 當我們執行 main.py 時，因為 my_utils 是被匯入的，
# 所以 __name__ 的值是 "my_utils"，以上測試區塊不會執行。

主題四：模組搜尋路徑、相對與絕對路徑匯入

A. Python 如何尋找模組？

當您寫下 import my_module 時，Python 解譯器會依序在「模組搜尋路徑」中尋找 my_module.py。順序通常是：

  1. 目前腳本所在的目錄。
  2. PYTHONPATH 環境變數中指定的目錄。
  3. Python 安裝時設定的標準函式庫路徑。

B. 套件 (Package) 與匯入方式

當專案變大時，我們會將相關模組放入資料夾來管理，這個資料夾就稱為「套件 (Package)」。一個資料夾要被視為套件，其中必須包含一個（可以是空的）__init__.py 檔案。

絕對路徑匯入 (Absolute Import)：從專案根目錄開始指定完整路徑，例如 from my_app.models import user。這是 PEP 8 官方推薦的作法，因為它最清晰。
相對路徑匯入 (Relative Import)：使用 . (目前目錄) 和 .. (上一層目錄) 來指定模組的相對位置，例如 from ..utils import formatter。它只能在套件內部使用。

前瞻性建議：作為初學者和未來的軟體工程師，應優先養成使用絕對路徑匯入的習慣。

結論與前瞻性觀點

模組與匯入系統是 Python 程式設計的支柱。一個優秀的開發者不僅知道如何編寫函式與類別，更重要的是知道如何將它們組織成結構良好、可維護、可複用的模-組與套件。

在機器學習專案中，您將會大量地匯入 NumPy, Pandas, Scikit-learn, PyTorch 等第三方函式庫，並將自己的程式碼拆分為多個模組。熟練掌握 import 系統，是您從小型腳本開發邁向大型專案工程的關鍵一步。
